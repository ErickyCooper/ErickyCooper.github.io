【算法思维札记 - 002】

算法思维札记
不只讲答案，专讲思路演进与迁移能力

本文定位： 字符串 + 滑动窗口

题目卡片

  • 题目：使二进制字符串变成交替字符串的最少操作次数
  • 难度：中等
  • 标签：字符串、滑动窗口、失配计数
  • 题意：给定二进制串 s，可进行两类操作：
    ① 把首字符移到末尾（可做任意次）；
    ② 反转任意字符（0 ↔ 1）。
    目标是在允许任意次类型 1 的前提下，最小化类型 2 的次数，使字符串变为交替串。
  • 示例：
    输入 s = "111000"，输出 2
    输入 s = "010"，输出 0
    输入 s = "1110"，输出 1

核心问题

这题本质是：在所有“旋转后长度为 n 的窗口”里，找与交替模板失配最少的那个窗口。

第一直觉

第一反应是“枚举每一种旋转，再计算改成交替串要翻转多少位”。

但这样会变成 O(n^2)：n 次旋转 × 每次 O(n) 比较，字符串长时会超时。

卡点

卡点 1：旋转是环状结构，直接逐次构造旋转串很重。

卡点 2：每次都从头统计失配，重复计算太多。

关键思考点

  • 把字符串扩展成 s2 = s + s，这样任意一次“首字符移到末尾”都对应 s2 里的一个长度为 n 的连续窗口；
  • 交替目标只有两种模板：0101... 和 1010...；
  • 用滑动窗口维护当前窗口相对两种模板的失配数 diff1、diff2；
  • 窗口每右移一格，只增量更新进出字符，不重算整段。

结论：遍历所有长度为 n 的窗口，取 min(diff1, diff2) 的最小值即答案。

最终解法（思路）

  1. 设 n = len(s)，构造 s2 = s + s；
  2. 遍历 s2，同步统计当前位置对两种交替模板的失配增量；
  3. 当窗口长度超过 n，移除左端字符对失配的贡献；
  4. 当窗口长度正好为 n，用 min(diff1, diff2) 更新答案。

代码（Go）

复杂度：时间 O(n)，空间 O(n)

思考

  1. 遇到“可旋转/环状”字符串，优先想到 s + s 把环展开成线性窗口。
  2. 遇到“最少修改成某模式”，可先把“修改次数”转为“失配次数”。
  3. 当目标模式数量很少（这里只有两种），可以并行维护多个计数器一次完成。

本题评分

  • 思维启发度：⭐️⭐️⭐️⭐️⭐️
  • 套路复用度：⭐️⭐️⭐️⭐️⭐️
  • 实现复杂度：⭐️⭐️⭐️☆☆
  • 面试价值：⭐️⭐️⭐️⭐️⭐️

一页总结

  1. 旋转不必真的转，s + s + 固定窗口就能覆盖所有情况。
  2. 翻转最少次数 = 与目标模板的最小失配数。
  3. 滑窗增量更新是这题从 O(n^2) 到 O(n) 的关键。
