算法思维札记
不只讲答案，专讲思路演进与迁移能力

本文定位： 字符串 + 构造 + 对角线法

题目卡片

  • 题目：找出不同的二进制字符串
  • 难度：中等
  • 标签：数组、字符串、构造、对角线法
  • 题意：给你一个长度为 n 的字符串数组 nums，其中每个字符串长度也为 n，且彼此互不相同。请返回任意一个长度为 n、且不在 nums 中的二进制字符串。
  • 约束条件：
    • n == nums.length
    • 1 <= n <= 16
    • nums[i].length == n
    • nums[i] 只包含 '0' 或 '1'
    • nums 中所有字符串互不相同
  • 示例：
    输入 nums = ["01","10"]，输出 "11"（"00" 也可）
    输入 nums = ["00","01"]，输出 "11"（"10" 也可）
    输入 nums = ["111","011","001"]，输出 "101"（"000" 等也可）

核心问题

这题本质是：构造一个字符串，让它和第 i 个字符串在第 i 位必定不同。

第一直觉

第一反应通常是：枚举所有长度为 n 的二进制串，再用哈希表判断是否存在。

这个思路可做，但显得“重”。因为总空间是 2^n，本题其实可以直接线性构造答案。

卡点

卡点 1：把问题当搜索题做，忽略了“只要任意一个答案”的构造空间。

卡点 2：只保证“不等于某几个字符串”，却没系统性保证“不等于所有字符串”。

关键思考点

  • 设构造串为 ans；
  • 对每个位置 i，令 ans[i] 与 nums[i][i] 取反；
  • 这样 ans 与第 i 个字符串在第 i 位必不同；
  • 因此 ans 不可能等于任何一个 nums[i]。

结论：按主对角线逐位取反，一次构造即可得到合法答案。

最终解法（思路）

  1. 初始化长度为 n 的字节数组 ans；
  2. 遍历 i = 0..n-1；
  3. 若 nums[i][i] == '0'，则 ans[i] = '1'，否则置为 '0'；
  4. 返回 string(ans)。

代码（Go）

复杂度：时间 O(n)，空间 O(n)

思考

  1. 当题目要求“构造一个不在集合中的元素”时，优先想构造法而不是暴力搜索。
  2. 对角线法的核心是“逐个对象设置一个专属差异位”，保证全局不相等。
  3. 这是典型的“从判定转构造”的思维升级。

本题评分

  • 思维启发度：⭐️⭐️⭐️⭐️⭐️
  • 套路复用度：⭐️⭐️⭐️⭐️☆
  • 实现复杂度：⭐️☆☆☆☆
  • 面试价值：⭐️⭐️⭐️⭐️☆

一页总结

  1. 这题不是找答案，而是“设计答案”。
  2. 对角线取反让结果与每个原串至少一位不同，从而全局排除。
  3. 一次线性遍历就能完成构造，简洁且稳定。
